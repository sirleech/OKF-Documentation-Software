# 02 · Architecture

How the system is designed: the overall system view, diagrams, per-component
docs, and Architecture Decision Records (ADRs).

## Contents

- `system-overview.md` — High-level description of the system architecture.
- [`diagrams/`](diagrams/) — Context, container, and sequence diagrams.
- [`components/`](components/) — Documentation for each major component.
- [`decisions/`](decisions/) — Architecture Decision Records and their index.

> Before changing any non-trivial subsystem, read its ADR in
> [`decisions/`](decisions/) — those records carry "don't reintroduce X"
> warnings for bugs that were already fixed once.
