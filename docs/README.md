# Documentation

Structured project documentation organized by domain. Each top-level folder
covers a distinct area of the software lifecycle. If a section doesn't apply to
this project, leave it out entirely.

## Sections

| Folder | Purpose |
|---|---|
| [`00-overview`](00-overview/) | Project vision, high-level architecture, and shared glossary |
| [`01-product`](01-product/) | Requirements, design assets, and roadmap |
| [`02-architecture`](02-architecture/) | System design, component docs, and Architecture Decision Records (ADRs) |
| [`03-development`](03-development/) | Local setup, coding guidelines, branching, and contribution guide |
| [`04-api`](04-api/) | API overview, authentication, endpoint references, and OpenAPI spec |
| [`05-testing`](05-testing/) | Testing strategy and docs for unit, integration, and end-to-end tests |
| [`06-deployment`](06-deployment/) | CI/CD pipelines, environment configs, infrastructure, and rollback procedures |
| [`07-operations`](07-operations/) | Monitoring, logging, alerting, incident response, and maintenance |
| [`08-security`](08-security/) | Security overview, auth/authz, data protection, threat model, and compliance |
| [`09-user-docs`](09-user-docs/) | End-user guides, tutorials, FAQ, and troubleshooting |
| [`10-release-notes`](10-release-notes/) | Per-version release notes and running changelog |

> Every folder contains a `README.md` so GitHub renders it as the default
> landing page when the folder is opened.
